# رفيقي — مساعد دراسي للمنهج الليبي

تطبيق ويب (React + خادم Express) يعمل تلقائياً مع أي مزوّد ذكاء اصطناعي متوفّر مفتاحه
في متغيرات البيئة: **Claude، Gemini، DeepSeek، OpenAI**، أو أي مزوّد آخر متوافق مع OpenAI.

## ⚠️ نقطة مهمة قبل أي شيء: لماذا فيه خادم خلفي (server.js)؟

في النسخة الأصلية كان الكود يستدعي `api.anthropic.com` **مباشرة من المتصفح**. هذا غير آمن
لموقع حقيقي: أي متغيّر بيئة يُستخدم داخل كود الواجهة الأمامية (frontend) يُحزَم داخل ملفات
JavaScript العامة التي يُحمّلها كل زائر — أي أن مفتاح الـ API سيكون مرئياً لأي شخص يفتح
"عرض مصدر الصفحة" أو تبويب Network في المتصفح، ويمكن سرقته واستخدامه على حسابك.

لذلك أضفنا `server.js`: خادم Express بسيط هو الوحيد الذي يقرأ مفاتيح الـ API من متغيرات
البيئة على الخادم، والواجهة الأمامية لا تتحدث إلا مع نقطة النهاية الخاصة بنا `/api/chat`
على نفس الموقع. المفتاح لا يغادر الخادم أبداً.

## كيف يختار المزوّد تلقائياً؟

عند بدء تشغيل الخادم، يبحث بالترتيب التالي عن أول مفتاح موجود في متغيرات البيئة:

1. `ANTHROPIC_API_KEY` → Claude
2. `GEMINI_API_KEY` أو `GOOGLE_API_KEY` → Gemini
3. `DEEPSEEK_API_KEY` → DeepSeek
4. `OPENAI_API_KEY` → OpenAI
5. `AI_API_KEY` + `AI_BASE_URL` → أي مزوّد آخر متوافق مع OpenAI (Groq, Mistral, OpenRouter...)

ضع متغيّراً واحداً فقط فيُستخدم تلقائياً، أو أكثر من واحد وأجبر الاختيار بمتغيّر
`AI_PROVIDER=anthropic|gemini|deepseek|openai|custom`. لا حاجة لتعديل أي كود — فقط
غيّر متغيرات البيئة في Render وأعد التشغيل.

يظهر شريط صغير أعلى التطبيق يوضح للطالب (ولك أثناء الاختبار) أي مزوّد متصل حالياً، أو
تنبيه واضح إن لم يُضبط أي مفتاح بعد.

> **ملاحظة دعم المرفقات:** الصور وملفات PDF (تبويب 📎) مدعومة بالكامل مع **Claude و Gemini**.
> أما **DeepSeek وOpenAI** (عبر مسار `chat/completions` المستخدَم هنا) فهي نصيّة فقط، لذا
> سيصل النموذج رسالة نصية توضح أن المرفق لم يُرسل بدل تجاهله بصمت.

## النشر على Render

1. ارفع هذا المجلد إلى مستودع GitHub.
2. على Render: **New → Web Service** واربطه بالمستودع.
3. إعدادات البناء والتشغيل:
   - **Build Command:** `npm install && npm run build`
   - **Start Command:** `npm start`
4. من **Environment**: أضف متغيّراً واحداً على الأقل من القائمة أعلاه (مثلاً `ANTHROPIC_API_KEY`).
5. Deploy. بعد انتهاء البناء سيعمل الموقع على الرابط الذي يعطيك إياه Render، ويعمل تلقائياً
   مع أي مزوّد ضبطت مفتاحه — دون لمس كود.

## التشغيل محلياً للتجربة

```bash
npm install
cp .env.example .env   # ثم ضع مفتاحاً واحداً على الأقل داخله

# في تيرمنال أول: شغّل الخادم (يقرأ .env عبر متغيرات النظام، أو استخدم dotenv/أداة مثل `direnv` إن أردت تحميل .env تلقائياً)
npm run dev:server

# في تيرمنال ثانٍ: شغّل واجهة Vite للتطوير
npm run dev
```

ثم افتح الرابط الذي يطبعه Vite (عادة `http://localhost:5173`).

## الإصلاحات المطبّقة على الكود الأصلي

1. **خلل حرج في `buildApiMessages`** — كانت رسائل المساعد (assistant) تُصنَّف خطأً بدور
   "user"، فيُرسَل للذكاء الاصطناعي تاريخ محادثة بالكامل بدور "user" متتالٍ، وهو ما يخالف
   شرط تناوب الأدوار في كل مزوّدات الذكاء الاصطناعي تقريباً ويُسقط الطلب بخطأ. تم تصحيحه.
2. **تخزين المرفقات الثقيلة في التخزين الدائم** — كان يُحفَظ كامل بيانات base64 للصور
   وملفات PDF (حتى 15 ميغا للملف) ضمن سجل المحادثة المحفوظ، ما قد يتجاوز حدود تخزين
   المتصفح ويفشل بصمت. الآن تُحفظ فقط بيانات المرفق الوصفية (الاسم والنوع) دون محتواه
   الثنائي، بينما يبقى المرفق الكامل متاحاً للذكاء الاصطناعي أثناء الجلسة الحيّة نفسها.
3. **فحص نوع الملف الهش** — كان يعتمد فقط على `file.type` الذي قد يكون فارغاً لملفات PDF
   أو صور صالحة على بعض المتصفحات/الأنظمة. أُضيف فحص احتياطي بامتداد اسم الملف.
4. **نمو مصفوفة الرسائل بلا حد في الذاكرة** — أُضيف سقف (120 رسالة) يحافظ على الرسالة
   الأولى وأحدث الرسائل، لمنع تضخّم الذاكرة في الجلسات الطويلة جداً.
5. (تحسين إضافي ضروري للنشر الحقيقي) **التخزين الدائم تحوّل من `window.storage`
   (متاح فقط داخل بيئة Claude.ai Artifacts) إلى `localStorage` الحقيقي في المتصفح**،
   وإلا لن يعمل حفظ التقدّم/الاختيارات/المحادثات إطلاقاً على موقع مستقل مثل هذا.
